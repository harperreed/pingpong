# GoReleaser Release Pipeline Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate pingpong's release pipeline to GoReleaser so `brew install harperreed/tap/pingpong` works and matches the rest of the tap, while keeping crates.io publishing.

**Architecture:** A tag push (`v*`) triggers a new `release.yml` with two jobs: a macOS-runner `goreleaser` job that builds both darwin binaries with native cargo, creates the GitHub release, and pushes `Formula/pingpong.rb` to `harperreed/homebrew-tap`; and an `ubuntu` `publish-crate` job that publishes to crates.io. The Homebrew formula is generated by GoReleaser (`.goreleaser.yaml`), not the hand-rolled shell script.

**Tech Stack:** Rust (cargo), GoReleaser v2 (open-source distribution, Rust builder, native cargo tool — no zig), GitHub Actions, Homebrew.

## Global Constraints

Copied verbatim from the design spec. Every task implicitly includes these.

- GoReleaser distribution is `goreleaser` (open-source), version constraint `~> v2`. The Rust builder is stable since v2.5.
- Release is **macOS-only**: targets `x86_64-apple-darwin` and `aarch64-apple-darwin`. No Linux/Windows release binaries.
- Build tool is native **`cargo`** (no zig / cargo-zigbuild).
- The binary/command name is **`pingpong`** everywhere (cargo build, cargo install, brew). The crate name on crates.io stays `pingpong-rs` (`[package].name`, unchanged).
- Homebrew tap: `harperreed/homebrew-tap`; formula is macOS-only; formula test is `system "#{bin}/pingpong", "--help"`.
- Archive name template: `pingpong_{{ .Version }}_Darwin_{x86_64|arm64}` (map GoReleaser's `amd64` → `x86_64`).
- Secrets already configured on the repo: `GITHUB_TOKEN` (auto), `TAP_GITHUB_TOKEN`, `CARGO_REGISTRY_TOKEN`.
- **Never** use `git commit --no-verify`. All pre-commit hooks must pass. The Rust hooks (`fmt`, `clippy`, `test`, `check`) only run when `.rs` files are staged.

## File Structure

- **Modify** `Cargo.toml` — add a `[[bin]]` target named `pingpong`.
- **Modify** `.github/workflows/ci.yml` — update `binary-name` values from `pingpong-rs` to `pingpong` / `pingpong.exe`.
- **Modify** `Makefile` — fix the `release` help-comment path (`pingpong-rs` → `pingpong`).

- **Delete** `scripts/update-homebrew-formula.sh` — replaced by GoReleaser `brews`.

### Task 2: Add `.goreleaser.yaml` and verify with a local snapshot

Create the GoReleaser config and prove it builds both darwin archives and generates a correct `pingpong.rb` **without publishing anything**, by diffing against an existing tap formula.

**Files:**
- Create: `.goreleaser.yaml`

**Interfaces:**
- Consumes: the `pingpong` binary name from Task 1 (`binary: pingpong`).
- Produces: `.goreleaser.yaml` driving the `goreleaser` job in Task 3; a verified-shape `Formula/pingpong.rb`.

- [ ] **Step 1: Install GoReleaser (prerequisite)**

Run:
```bash
brew install goreleaser
goreleaser --version
```
Expected: GoReleaser v2.x installed.

- [ ] **Step 2: Write the failing check**

Run:
```bash
goreleaser check
```
Expected now: **FAIL** — no `.goreleaser.yaml`/`.goreleaser.yml` found.

- [ ] **Step 3: Create `.goreleaser.yaml`**

```yaml
# ABOUTME: GoReleaser config building macOS binaries for pingpong and
# ABOUTME: publishing its Homebrew formula to harperreed/homebrew-tap.
version: 2

project_name: pingpong

builds:
  - id: pingpong
    builder: rust
    binary: pingpong
    tool: cargo
    flags:
      - --release
    targets:
      - x86_64-apple-darwin
      - aarch64-apple-darwin

archives:
  - id: pingpong
    formats: [tar.gz]
    # GoReleaser renders Go-style arch (amd64/arm64); map amd64 -> x86_64 to
    # match the existing tap asset names (Darwin_x86_64 / Darwin_arm64).
    name_template: 'pingpong_{{ .Version }}_Darwin_{{ if eq .Arch "amd64" }}x86_64{{ else }}{{ .Arch }}{{ end }}'

checksum:
  name_template: "checksums.txt"

brews:
  - name: pingpong
    repository:
      owner: harperreed
      name: homebrew-tap
      token: "{{ .Env.TAP_GITHUB_TOKEN }}"
    homepage: "https://github.com/harperreed/pingpong"
    description: "A beautiful TUI ping utility for monitoring network connectivity"
    license: "MIT"
    test: |
      system "#{bin}/pingpong", "--help"

release:
  github:
    owner: harperreed
    name: pingpong
```

- [ ] **Step 4: Validate the config**

Run:
```bash
goreleaser check
```
Expected: PASS — "1 configuration file(s) validated". If the installed GoReleaser flags a renamed/removed field (e.g. `formats` vs `format`, or a `brews`→`homebrew_casks` deprecation), adjust the field to the installed version's schema and re-run. Keep `brews` (a Formula), not a cask, to match the existing tap formulae — a deprecation *warning* is acceptable and non-fatal.

- [ ] **Step 5: Build a snapshot (no publish) — the verification gate**

`TAP_GITHUB_TOKEN` is referenced by the config template, so provide a dummy value for the snapshot (nothing is pushed in snapshot mode):
```bash
TAP_GITHUB_TOKEN=dummy goreleaser release --snapshot --clean
```
Expected: PASS — `dist/` contains two `pingpong_*_Darwin_x86_64.tar.gz` / `_Darwin_arm64.tar.gz` archives, a `checksums.txt`, and a generated formula (under `dist/homebrew/Formula/pingpong.rb` or similar).

- [ ] **Step 6: Verify the generated formula's shape**

Run:
```bash
find dist -name 'pingpong.rb' -exec cat {} \;
```
Expected: a `class Pingpong < Formula` with `version`, macOS `url`/`sha256` blocks for both arches, `bin.install "pingpong"`, and `test do system "#{bin}/pingpong", "--help" end`. Confirm it structurally matches an existing tap formula (compare against `toki.rb`: `gh api repos/harperreed/homebrew-tap/contents/Formula/toki.rb --jq .content | base64 -d`).

- [ ] **Step 7: Confirm the snapshot artifacts are not committed**

`dist/` must be git-ignored. Run:
```bash
grep -qxF 'dist/' .gitignore || echo 'dist/' >> .gitignore
git status --short
```
Expected: `.goreleaser.yaml` shows as a new file; `dist/` does not appear as untracked.

- [ ] **Step 8: Commit**

```bash
git add .goreleaser.yaml .gitignore
git commit -m "ci: add GoReleaser config for macOS build and Homebrew formula"
```

### Task 3: Rewrite `release.yml` and delete the formula script

Replace the entire release workflow with the two-job GoReleaser pipeline and remove the now-dead hand-rolled formula script.

**Files:**
- Rewrite: `.github/workflows/release.yml`
- Delete: `scripts/update-homebrew-formula.sh`

**Interfaces:**
- Consumes: `.goreleaser.yaml` from Task 2; the `pingpong` binary from Task 1; repo secrets `GITHUB_TOKEN`, `TAP_GITHUB_TOKEN`, `CARGO_REGISTRY_TOKEN`.

- [ ] **Step 1: Replace `.github/workflows/release.yml` with:**

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  goreleaser:
    name: GoReleaser (macOS)
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: x86_64-apple-darwin, aarch64-apple-darwin

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: '~> v2'
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAP_GITHUB_TOKEN: ${{ secrets.TAP_GITHUB_TOKEN }}

  publish-crate:
    name: Publish to crates.io
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Publish to crates.io
        uses: katyo/publish-crates@v2
        with:
          registry-token: ${{ secrets.CARGO_REGISTRY_TOKEN }}
```

- [ ] **Step 2: Delete the hand-rolled formula script**

Run:
```bash
git rm scripts/update-homebrew-formula.sh
```

- [ ] **Step 3: Lint the new workflow**

Run:
```bash

```
Expected: PASS — no errors.

- [ ] **Step 4: Confirm the deprecated actions and dead script are gone**

Run:

- [ ] **Step 5: Commit**

